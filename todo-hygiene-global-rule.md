# Todo / 待辦文件衛生（全域規則）

## 規則

維護 `tasks/todo.md`（或任何待辦清單文件）時，**每次開始新任務、或被要求「把目前待辦寫進去」之前，先確認文件裡有沒有舊任務殘留的項目，不要直接在後面疊加**。

這條規則存在的理由：曾經發生過——重開 session 前被要求把當下待辦寫進 `todo.md`，但沒有先清掉裡面舊的、屬於另一個已經結案的功能（LEAPS）的項目，導致同一份文件裡新舊內容混在一起。新 session 接手時讀到這份文件，誤以為裡面列的舊項目（例如「履約價 input bug 待修復」「Phase F 尚未實作」）是目前真實狀態，但這些其實早就已經確認完成，只是文件沒清過。最後要靠額外一輪對話釐清「這是新舊堆疊，不是真的矛盾」，浪費了原本不需要花的時間。

## 具體做法

1. **單一任務的進度，單一事實來源**：如果某個功能/任務已經有專屬的規格文件在追蹤狀態（例如本專案 LEAPS 功能的 `leaps-call-recommendation-spec.md` 開頭那段「接手前必讀」快照），`todo.md` 不要重複維護那份進度，最多只能放一行指向那份文件，不要展開複製內容——複製出來的內容一旦原文件更新，這份副本馬上就過期。
2. **不同任務的待辦不要混在同一份 todo 裡累積**：每次開始新任務寫待辦清單前，先檢查文件裡現有內容是屬於哪個任務的，跟當前任務無關的舊項目要先清掉或移到別處歸檔，不要讓文件變成所有任務的待辦堆疊在一起、誰也分不清哪些還有效。
3. **重開 session 前要求「把目前待辦寫進去」時**，先確認這個動作的範圍：是「只記錄當前這個任務接下來要做的事」，還是「完整覆寫整份文件」。如果是前者，預設行為應該是先清空跟當前任務無關的舊內容，而不是無條件 append。

## 驗收標準

- [ ] 任何待辦清單文件被更新時，沒有把不同任務、不同時期的項目疊加在一起累積。
- [ ] 已經有專屬規格文件追蹤進度的功能，待辦清單裡沒有重複維護一份內容，只有指向專屬文件的指引（如果需要的話）。
